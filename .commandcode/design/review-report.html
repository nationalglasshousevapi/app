// PRIORITY ISSUES
P0

Generic copy undermines confidence

"Create something for your customer" and "Start with the document type, then select a saved customer to fill their details quickly." — these read like chatbot output, not a tool that produces financial documents. The heading could belong to any app.

FIX: Replace with specific page title ("New Invoice", "New Quotation"). Use actual doc type. Remove procedural instructions from subtitles — users know how forms work.
P0

Bevelled clip-path clips content

`.card` uses `clip-path: polygon(0 0, calc(100% - 14px) 0, 100% 14px, 100% 100%, 0 100%)` which cuts 14px off the top-right corner. Right-aligned content, close buttons, step badges, and status pills near the corner get visibly cropped.

FIX: Move card inner content away from the clipped corner, or add padding-right to compensate for the cut region. Alternatively, keep the bevel but ensure no critical content lives in the top-right 14px zone.
P1

No animation on shipping fields toggle

The "Ship to same as billing" checkbox instantly shows/hides shipping fields. No opacity, height, or scaleY transition. For a daily operator who creates invoices repeatedly, this blink adds friction every time.

FIX: Add a 150-200ms opacity + max-height transition on the shipping section wrapper. Use `grid-template-rows: 0fr / 1fr` or a max-height toggle.
P1

Page headings lack presence

`.page-title` is `text-2xl` (22px). On a 1440px viewport, the primary heading competes with body text for weight. The compressed scale makes dashboard stat cards (with `text-3xl` values) visually outrank the page title.

FIX: Bump `.page-title` to 1.75rem (28px) or `text-3xl` (26px). Increase the spread between heading levels.
P2

Empty states are text-only

"No documents found" in gray center text. No illustration, no guidance, no action prompt. Compare to the dashboard which says "No invoices yet — create your first one to see sales here." — that's the right energy.

FIX: Give every empty state a message that names what goes here, why it matters, and the action to fill it. Use the dashboard's copy as the template.
P2

Line items breakpoints use `lg:` instead of `md:`

Column headers in LineItemsEditor appear at `lg:` (1024px). With the sidebar taking 256px, the content area at `md:` (768px) is ~512px — enough for table headers. Tablet landscape users see a stacked form unnecessarily.

FIX: Change the column header visibility from `hidden lg:grid` to `hidden md:grid`. Adjust mobile label visibility accordingly.
P2

Missing palette intermediates and frost chroma

Brand palette jumps from 100 to 500 to 600 to 700 to 900 — no 200, 300, 400, 800. Forces `brand-50` to do double duty. Frost (`#F6F8F8`) is a pure neutral with no tint toward the brand teal.

FIX: Add brand-200, brand-300, brand-400, brand-800. Tint frost with OKLCH chroma 0.01 at hue 190° — invisible as a color but ties canvas to brand.
